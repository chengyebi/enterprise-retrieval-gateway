# AGENTS.md

## 维护规则

- 尽量少问问题；除非涉及密钥、付费服务、删除大量代码、破坏性操作、公开 API 语义大改，否则自行判断并继续。
- 不做大重构，不引入大型依赖，保持 C++17。
- 不提交任何 secrets、真实令牌、数据库密码或本地环境配置。
- CMake 和 Makefile 的源码列表需要保持一致。
- 新增或修改的代码注释尽量使用中文；类名、函数名、文件名和 JSON 字段名保持英文。
- README、docs、web 页面文案、按钮和错误提示尽量使用中文。

## 安全规则

- 不得削弱 ACL。
- 未知用户、权限缺失、ACL 异常、backend 异常默认 fail-closed。
- 请求中的 `project_ids`、`document_types` 等过滤条件只能进一步收窄结果，不能放宽服务端 ACL。
- malformed JSON、缺失必填字段和非法边界值必须返回清晰错误，不能静默当作空请求。
- `top_k` 必须显式限制在安全范围内，`0`、负数和过大值不能导致异常放大或默认绕过。
- unknown fields 默认忽略。
- API 错误响应不要泄露内部实现细节、堆栈、后端细节或敏感配置。

## 测试规则

- 修改 API/JSON/ACL 边界时，必须补充回归测试。
- P0：未授权文档泄露、ACL fail-closed 被破坏、构建或测试失败。
- P1：malformed JSON 被静默接受、必填字段缺失错误不清晰、`top_k` 边界处理不安全、unknown fields 不兼容、错误响应泄露内部细节。
